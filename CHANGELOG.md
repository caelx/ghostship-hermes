# Changelog

All notable changes to this project will be documented in this file.

## Unreleased

- Pinned Hermes release updated to `v2026.4.8`.
- Fixed Hermes release automation so scheduled pin bumps also update the `hermes-agent` flake input and lockfile, then explicitly dispatch the image publish workflow instead of relying on an Actions-authored `push` event.

- Reworked the Hermes image toward a minimum-system runtime contract by converging the mutable `hermes` user toolchain on boot and daily refresh (`hermes`, `git`, `curl`, `jq`, `nix`, `ripgrep`, `node`, `npm`), managing `codex`, `gemini`, and `opencode` in a persisted npm layer, wrapping Hermes so the Nix-packaged `agent-browser` satisfies both runtime and `hermes doctor`, wiring per-profile `.env` files as the operator-facing source of truth, auto-starting the dashboard/router/profile gateways after bootstrap, auto-restarting gateways when profile `.env`/`config.yaml`/`auth.json`/`SOUL.md` change, switching dashboard/browser terminals to `/workspace`, and aligning ttyd to the dashboard's blue theme tokens.

- Disabled in-container `resolvconf` in the Hermes image so the NixOS container preserves Podman-provided `/etc/resolv.conf` and external DNS resolution works for router upstreams such as OpenRouter and OpenCode.

- Restored dashboard compatibility for older clients by serving the legacy `/api/profiles.json` payload alongside the current `/api/status` API so stale edge/browser bundles stop hard-failing during Hermes dashboard rollouts.

- Added `ghostship-n8n`, a JSON-first CLI for the full official n8n Public API, mirrored the bundled upstream OpenAPI contract into `docs/api/n8n-openapi.json`, and documented the new utility in `docs/api/n8n.md`.

- Added `ghostship-chaptarr`, a JSON-first CLI for every Chaptarr public API endpoint, mirrored the upstream Readarr-based OpenAPI contract into `docs/api/chaptarr-openapi.json`, and documented the env/command contract in `docs/api/chaptarr.md`.

- Added a vendored `agent-browser` package to the image, widened the shared Hermes browser env passthrough for Browserbase, Browser Use, Camofox, and one optional `BROWSER_CDP_URL` passthrough, kept Chrome/Chromium out of the image, and documented that Hermes local browser mode defaults to `agent-browser` while CDP attachment remains a single interactive `/browser connect` target rather than a declarative multi-CDP config.
- Added a shared Hermes timezone default of `Pacific/Honolulu` to the three-profile scaffold so scheduling, reminders, and session behavior align with the intended operator local time.
- Added shared Hermes `agent` defaults to the three-profile scaffold (`max_turns = 110`, `reasoning_effort = "high"`, `verbose = false`) to better fit long-running agentic profiles.
- Enabled Hermes Holographic memory in the three-profile scaffold by setting `memory.provider = holographic`, keeping built-in memory and user-profile memory enabled, setting `nudge_interval = 10` and `flush_min_turns = 6`, and using explicit local `plugins.hermes-memory-store` defaults (`db_path = $HERMES_HOME/memory_store.db`, `auto_extract = false`, `default_trust = 0.5`).
- Added shared Hermes transcript compression defaults to the three-profile scaffold (`enabled = true`, `threshold = 0.50`, `target_ratio = 0.25`, `protect_last_n = 20`) to keep long-lived sessions manageable.
- Added an explicit shared Hermes `session_reset` policy with `mode = "none"` plus placeholder `idle_minutes = 1440` and `at_hour = 4`, so long-running sessions stay continuous unless the policy is changed later.
- Added shared Hermes browser defaults to the three-profile scaffold (`cloud_provider = "local"`, `inactivity_timeout = 120`, `command_timeout = 30`, `record_sessions = false`).
- Set shared Hermes approvals to `mode = "off"` in the three-profile scaffold for a trusted long-running agent runtime.
- Added shared Hermes security defaults to the three-profile scaffold, enabling secret redaction and Tirith with a fail-open posture plus a disabled website blocklist scaffold.
- Enabled shared Hermes checkpoints in the three-profile scaffold with `max_snapshots = 50` for mutation rollback safety.
- Added shared Hermes streaming defaults to the three-profile scaffold (`enabled = true`, `transport = "edit"`, `edit_interval = 0.3`, `buffer_threshold = 40`).
- Added shared Hermes display defaults to the three-profile scaffold (`compact = false`, `tool_progress = "new"`, `background_process_notifications = "result"`).
- Explicitly disabled Hermes STT in the three-profile scaffold with `stt.enabled = false`.
- Explicitly disabled Hermes artificial response delay in the three-profile scaffold with `human_delay.mode = "off"`.
- Added baseline Discord gateway settings to each scaffolded Hermes profile (`require_mention`, `auto_thread`, `reactions`, `group_sessions_per_user`) and mapped profile-specific Discord env vars into the standard Hermes Discord env names so a shared general channel can stay mention-only while each role channel is free-response; distinct icons still require separate Discord bot applications rather than a Hermes profile setting.
- Taught `ghostship-hermes-router` to accept the Hermes-facing `OPENCODE_GO_API_KEY` env var as an alias for `OPENCODE_API_KEY`, so the current Hermes Minimax fallback credential name can be reused without a second OpenCode env var.
- Cleaned up the repo env-var docs to separate the current Hermes scaffold requirements (`OPENCODE_GO_API_KEY`, `GOOGLE_AI_STUDIO_API_KEY`, optional `BWS_ACCESS_TOKEN`) from the router-only OpenRouter/OpenCode runtime env surface, and corrected the router alias env names to match the live `auxiliary` / `coding` / `agentic` / `vision` / `tts` alias set.
- Set the scaffolded Hermes main model lane to `openai-codex/gpt-5.4` with Hermes-native fallback to `opencode-go/minimax-m2.7`, and pinned all Hermes auxiliary tasks to Gemini 3.1 Flash-Lite Preview through Google's OpenAI-compatible Gemini endpoint using `GOOGLE_AI_STUDIO_API_KEY` while leaving TTS unconfigured.
- Re-enabled packaged docs, man pages, info pages, and NixOS docs in the Hermes image so in-container agents can inspect local reference material.
- Simplified the three-profile Hermes scaffold to stop generating per-profile `.env` files, keep only the shared Hermes `.env` for the handful of Hermes-native provider secrets the scaffold currently needs, let services inherit runtime container env directly, and include the `bws` Bitwarden Secrets Manager CLI in the default image package set.
- Replaced the old two-profile Hermes image scaffold with a Nix-first `assistant` / `operations` / `supervisor` profile matrix, switched the sticky default profile to `assistant`, kept the root Hermes config minimal, generated profile config/env/skill/service metadata from that matrix, and kept shared plus per-profile skills on the existing copy-once runtime seeding path.
- Restored the pinned upstream `gws` Google Workspace CLI to the default Hermes image `PATH`, exposed it as a flake package for both supported publish architectures, updated runtime docs and validation for the new default, and kept Google Workspace support explicitly CLI-only without vendored or seeded skills.
- Hardened router/image validation by caching SQLite-backed router state reads so `/v1/models` no longer times out on live inventories, recreating the router DB path on connect, putting the validation toolchain on the system PATH for root-side image checks, and updating the dashboard and persistence smoke scripts for the current alias set with optional-empty `tts`.
- Restored opt-in Hermes skill staging for both shared and per-profile skill trees via `/home/hermes/seeds/shared/skills/<skill>` and `/home/hermes/seeds/profiles/<profile>/skills/<skill>`, with bootstrap copying each missing skill directory once and leaving any existing Hermes-managed skill directory untouched on later boots.
- Enforced free-only routing in `ghostship-hermes-router`, removed the image's hardcoded alias shortlists so startup bucketing now comes from persisted state plus live discovery, kept startup unavailable until the first discovery pass finishes when no persisted state exists, preferred an OpenCode Zen worker for dynamic bucketing with OpenRouter fallback, required `coding` and `auxiliary` routed models to be tool-calling capable with text-only outputs when provider metadata exposes those capabilities, allowed free `vision` models without tool calling when they accept image or video input and return text, limited free `tts` routing to speech-style audio models and explicitly excluded music generators such as Lyria, increased alias-family and recency weighting while reducing raw name-signal and provider weight, fixed family matching so exact id/name hits beat description-only hints, replaced the old absolute family-gap scoring with relative rank bonuses among the families that are actually present, added only a modest global size-rank bonus for parameterized `coding`/`agentic`/`vision` models, narrowed the smaller-model penalty so it only applies when a larger sibling exists in the same family or inferred subfamily, kept `auxiliary` smaller-is-better, enriched matched Zen models with public OpenRouter description/created/modality metadata, and normalized provider-prefixed shortlist pins back to real backend model ids before dispatch.
- Fixed the Hermes router-primary image config to write `model.provider = auto` with the local router `base_url`, because upstream Hermes otherwise prefers its OpenRouter credential pool and bypasses the configured local endpoint whenever `OPENROUTER_API_KEY` is present.
- Disabled the flaky `python3.11-websockets` unittest suite in the shared router/dashboard Python package set so the native `aarch64-linux` image publication build can complete again on GitHub Actions.
- Corrected the multi-arch image publish workflow so the amd64 leg stays on `ubuntu-24.04`, the arm64 leg builds on `ubuntu-24.04-arm`, x86-host validation remains `nix eval`-only for arm64 wiring, and the docs now spell out that Docker QEMU plus Nix `extra-platforms` are not enough to publish the native arm64 image artifact.
- Switched the Hermes image to a router-primary runtime with root `coding`, `operations` `coding`, and `coder` `coding` over `http://127.0.0.1:8788/v1`, while exposing `auxiliary`, `coding`, `agentic`, `vision`, and `tts` router aliases, made the dashboard report generic root/profile endpoint config with live local-router enrichment, updated smoke and persistence validation to prove the router path, and moved router startup refresh/ranking off the listener-critical path so persisted rankings remain usable while fresh rankings regenerate in the background.
- Made `packages/hermes-dashboard` the canonical packaged dashboard artifact, exposed it as a direct flake package, removed the stale image runtime asset-root seam, updated the dashboard smoke test and docs to the current glass UI contract, wrapped the packaged entrypoint with the `ttyd` runtime it needs for `+` terminal launches, hardened ttyd port selection so stale listeners do not hijack new dashboard sessions, fixed the ttyd HTTP and websocket proxying so embedded terminals render and stay connected correctly, replaced the abandoned MMX styling with a modern glass terminal layout, and retuned the web UI into a darker Hermes-tailored shell with the Hermes logo, sharper geometry, and a square terminal stage.
- Tightened Hermes custom-provider compatibility for `ghostship-hermes-router` by accepting `API_SERVER_HOST` and `API_SERVER_PORT`, allowing `OPENAI_API_KEY` as the router bearer-token alias, and documenting the `custom_providers` base URL contract at `http://127.0.0.1:8788/v1`.
- Expanded `ghostship-hermes-router` again with Hermes-compatible `/health` and `/v1/health`, richer `chat/completions` SSE that preserves reasoning, tool-call, and usage deltas, SDK-compatible `responses.stream(...)`, richer stored `responses` objects, broader router contract tests, and a router package bump to `0.3.0`.
- Expanded `ghostship-hermes-router` with rolling model and provider health, Prometheus-style `/metrics`, provider-wide cooldown and recovery, free-model background ranking and shortlist reranking, durable override storage for disablement, weights, and alias pinning, richer ranking and provider debug endpoints, and runtime wiring for the new router tuning env.
- Added the new `packages/hermes-router` `FastAPI`/`uvicorn` service with stable alias endpoints, SQLite-backed route state, background inventory refresh, model-level routing and failover, best-effort first-text timing, OpenRouter plus OpenCode Zen inventory support, mixed Zen endpoint-family normalization, and free-model-assisted bucketing, then wired it into the Hermes image as `ghostship-hermes-router.service`.
- Added repo docs for the free-first model router plan and Hermes Nix setup notes, and gitignored the repo-local `.nix-local-store/` build cache so local validation state does not show up as source changes.
- Split the image build contract into an explicit publishable `ghostship-hermes-image` bundle and a separate `ghostship-hermes-rootfs` tarball, added `scripts/export_publishable_image.sh` as the shared Docker materialization helper, updated the GHCR publish workflow and dashboard smoke test to use that bundle, and kept the full persistence validation on the low-level rootfs path.
- Rebuilt the Hermes image around the upstream Hermes NixOS module with `HERMES_HOME=/home/hermes/.hermes`, `HOME=/home/hermes`, a dedicated `hermes` user at `3000:3000`, a persisted whole-home volume, and a lean default package set that keeps only Hermes, runtime Nix support, the dashboard stack, and the `ghostship-*` utilities.
- Removed the remaining Ghostship workstation baggage from both the image and the repo tree, including repo-managed skills, vendored Google Workspace assets, `feed`, `honcho-ai`, the old `rootfs`, and the old workstation seed payloads.
- Reworked the dashboard into a darker multi-terminal tab UI that keeps the old Hermes logo, removes the old inspect panel, serves directly from the controller process, opens each browser terminal at `/home/hermes`, creates focused tabs immediately with a loading state, labels sessions from the shell cwd or current command, allows unlimited ephemeral `ttyd` sessions, sandboxes the terminal iframe, and returns to a blank home state when all tabs are closed.
- Replaced the old bootstrap profiles with declarative `operations` and `coder` profiles, kept both gateways running through repo-managed systemd units, and made `operations` the sticky default CLI profile.
- Wired the bootstrap flow to consume `OPENROUTER_API_KEY` and `OPENROUTER_TEST_MODEL` from the runtime environment so both declared profiles come up with the same test provider configuration.
- Updated the image tests and validation flow to prove the final runtime contract, including gateway boot, `/home/hermes` persistence, multi-profile layout, multi-terminal dashboard behavior, persisted `/nix` installs, and later-installed tool state persistence across container replacement.
- Fixed the dashboard terminal websocket path so tab switching keeps the live `ttyd` session attached instead of falling into ttyd's reconnect overlay.
- Reworked the dashboard home state into a runtime environment console with live profile and provider metadata, home navigation on the Hermes logo, and a flatter Hermes-site-inspired dark UI treatment.
- Collapsed model namespaces under their transport provider in the dashboard, rendered configured models as provider subcards, renamed the home header to `ghostship-hermes`, and made the floating terminal close button a clearer red destructive control.
- Shifted the floating terminal close button farther left so it keeps the same top rhythm while staying clear of the ttyd scrollbar.
